Git init

Git init is the act of registering this current code base.

Git add

Git add is how we stage the intergration of our current code base into our current list of snapshots

Git status

Git status allows us ti see the changes we are going to make

Git commit

Git commit; finally we can commit those changes we added to our last snapshot giving us a new snapshot of our code with whatever new changes were made

Git remote add

Git remote add is used to set up a online repository where we can place our current code. This is where the outside world can see our code base.

Git push

Git push is how we get our code to Github, an excellent place to store all of our repositories. Other developers might look at our existing projects to see how we appraoched problems, pull down our code and add new features or even update our code base with features they wanted to add
